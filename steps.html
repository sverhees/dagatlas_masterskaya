Step by step instruction

On this page you can find information about what to do after choosing a feature and how to adjust your data according to the TALD format.

Step 1 - Create a folder

Make a folder for your feature in the archive and copy the feature template in it.

Step 2 - Add data

For each language fill all required columns (about the meaning of the columns see Feature datasets > List of columns).

  • We collect data from descriptive literature, which you can find in our library.

    • In our literature database you can find information about the idioms represented in the sources (sheet source type and idiom, columns idiom and type).

  • If you use a source that is not in the library yet:

    • Add information about the source to the literature database (see Feature datasets > Adding new literature). Do not forget to add the source under both tabs!

    • When you fill the idiom column on the sheet source type and idiom, add a screenshot with information about the source of the data under the language name in the doc with screenshots. Before the screenshot, write the bibtexkey from the main tab (to make the search easier it should be in the Heading 2 format) and the page number.

    • Upload the source to one of the folders in the library depending on the source type (grammars, grammar sketches, dictionaries or other). Name the file with bibtexkey and title from the main tab, for example: abdullaev1954_Грамматика даргинского языка.

Step 3 - Write a chapter

Write an introductory chapter about your feature (see Writing a chapter for more details about the structure and see an Rmarkdown cheatsheet here).

  • If you refer to any sources in your text, then copy the reference template and write the references in it (see Writing a chapter > Literature references).

  • If you use a glossed example in the text, use three non-alligned rows: transcription, glosses (with grammatical glosses in CAPS) and translation, like in (1). It will be aligned after the upload to the site:

    1. Standard Avar [@bokarev1949: 165]

      di-da-sa ɬik’-a-w qazaq du-je=gi šːʷ-ela-r=in

      I-SUO-EL good-ADJZ-M worker you.SG-DAT=ADD get-FUT-NEG=EMPH

      ‘You will also not get a better farm worker than me.’

Step 4 - Check your files

By the end of the work, your folder in the archive should have your text (in .Rmd format), the reference table for your text (.csv or .tsv) and your feature table.
